Applied methods/ regression vs the p-value
Comparing two groups

Is running Vol = b0 + b1·UserType + e and checking whether b1 is significant the right move?

Yes for the estimate, no for the significance test. b1 is exactly the difference in means. The p-value beside it answers a question whose answer is already known.

1b1 is the difference in means. That part is right.

With UserType coded 0/1 (0 = Type, 1 = DoubleClick), the regression is a two-group comparison:

$$\text{Vol} = b_0 + b_1\,\text{UT} + e, \qquad b_0 = \overline{\text{Vol}}_{\text{Type}}, \qquad b_1 = \overline{\text{Vol}}_{\text{DoubleClick}} - \overline{\text{Vol}}_{\text{Type}}.$$

So the coefficient is the number she wants. The estimate is fine. Two things around it are not: the significance test is a foregone conclusion, and the default standard error assumes equal variance.

3Why the significance is preordained

The t-statistic is about the effect divided by its SE, and the SE shrinks like $1/\sqrt{n}$, so $t$ grows like $(\text{effect})\times\sqrt{n}$. With tens of thousands of rows per group, any real difference is multiplied into a huge $t$ and a p-value that underflows to zero. Hold the gap fixed and drag the sample size up: the p-value crashes while b1 barely moves. Her two channels are obviously not the same distribution, so a test of "are they identical" can only say yes.

4The equal-variance trap, and when it bites

Plain lm() reports the pooled SE, which assumes one shared variance. The honest version is Welch, which uses each group's own variance, and for a single 0/1 regressor the robust HC2 SE equals it exactly. One subtlety: when the groups are the same size, pooled and Welch coincide, so the distinction looks harmless. Drag the size ratio off 1 and watch them separate. Her channels are close to equal size, so the numbers nearly agree, but the robust SE is the safe default: it costs nothing when sizes match and is correct when they do not.

5What to actually run

# b1 is the difference in means, either way:
m <- lm(Volume ~ UserType)

# Correct SEs even under unequal variance (HC2 == Welch for a 0/1 regressor):
library(sandwich); library(lmtest)
coeftest(m, vcov = vcovHC(m, "HC2"))
coefci(m,  vcov = vcovHC(m, "HC2"))     # report THIS interval, not the star

# Or the one-liner that is already Welch by default:
t.test(Volume ~ UserType)

Report b1 with its Welch interval as an effect size. Then say the useful thing: the difference is roughly multiplicative, DoubleClick runs about 5 to 7 times Type across the whole distribution.
